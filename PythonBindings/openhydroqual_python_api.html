Python API Reference

Python bindings for the OpenHydroQual aquifolium simulation engine. Build, configure, solve, and persist environmental models programmatically — enabling calibration workflows, sensitivity analysis, and reinforcement learning.

import import openhydroqual_py as ohq
Quick Start
Minimum working example — a 1D unconfined groundwater model.
1
Minimal 1D Groundwater Model

Load templates → build cells & links → finalize → solve

import openhydroqual_py as ohq

# ── 1. Create system and set paths ──────────────────────────────
system = ohq.System()
system.set_default_template_path('/path/to/resources/')
system.set_working_folder('/path/to/project/')
system.set_silent(False)

# ── 2. Load component templates ─────────────────────────────────
system.load_template('main_components.json')
system.add_template('unconfined_groundwater.json')

# ── 3. Simulation settings ───────────────────────────────────────
system.set_system_value('simulation_start_time', '0')
system.set_system_value('simulation_end_time',   '90')
system.set_system_value('initial_time_step',     '0.01')
system.set_system_value('write_interval',        '100')

# ── 4. Build blocks ──────────────────────────────────────────────
for i in range(1, 11):
    system.create_block(
        name=f'cell({i}:1)',
        type='Unconfined Groundwater cell',
        properties={
            'area':                  '5000',
            'hydraulic_conductivity': '5',
            'piezometric_head':      str(10.0 - (i-1) * 0.1),
            'porosity':              '0.3',
            'specific_yield':        '0.2',
        }
    )

# ── 5. Build links ───────────────────────────────────────────────
for i in range(1, 10):
    system.create_link(
        name=f'link({i}:{i+1})',
        type='UC_groundwater_link',
        from_block=f'cell({i}:1)',
        to_block=f'cell({i+1}:1)',
        properties={'length': '50', 'width': '100'}
    )

# ── 6. Finalize, solve, write ────────────────────────────────────
system.finalize()
system.solve()
system.write_outputs()
Workflow
Every model follows the same lifecycle. Steps must be performed in order.
ℹ
Order matters. Templates must be loaded before blocks/links are created (they define what types exist). finalize() must be called before solve(). Properties set after finalize() are not reflected in the solved state.
1 System() — Create instance, set paths & silence flag
2 load_template() / add_template() — Register component types from JSON
3 set_system_value() — Configure time range, timestep, output interval
4 create_block() / create_link() — Populate the model graph
5 finalize() — Wire parents, pointers, expressions, initial values
6 solve() — Run time integration
7 write_outputs() / save_to_script() / save_to_json() — Persist results

Templates
Templates are JSON files that define the available block and link types (their properties, units, expressions). They must be loaded before any create_block or create_link calls.
load_template(filename: str) → bool
setup
Loads the primary component template. Clears any previously loaded templates. Accepts full path or filename (resolved against default_template_path).
ParameterTypeDescription
filenamestrPath or filename of the JSON template (e.g. 'main_components.json')
python
system.load_template('/path/to/resources/main_components.json')
⚠
Always call load_template before add_template. Calling it again clears all existing templates.
add_template(filename: str) → bool
setup
Appends an additional template file to the existing MetaModel. Use to add domain-specific component libraries (groundwater, wastewater, etc.).
ParameterTypeDescription
filenamestrPath or filename of the JSON template to append
python — typical template loading sequence
RESOURCES = '/home/user/OpenHydroQual/resources/'
system.load_template(RESOURCES + 'main_components.json')
system.add_template(RESOURCES + 'unconfined_groundwater.json')
# For wastewater models:
# system.add_template(RESOURCES + 'wastewater.json')
# system.add_template(RESOURCES + 'mass_transfer.json')

Creating Blocks
Blocks are the computational nodes of the model (aquifer cells, reactors, reservoirs, fixed-head boundaries, etc.). Types must exist in a loaded template.
create_block(name: str, type: str, properties: dict[str,str] = {}) → bool
build
Creates a new block node in the model. The type string must match a block definition in the loaded templates. Properties are key-value pairs where all values are strings (even numeric ones).
ParameterTypeDescription
namestrUnique identifier for this block. Names with colons and parentheses are valid, e.g. 'cell(1:1)'
typestrBlock type as defined in the template JSON (case-sensitive)
propertiesdictOptional property overrides. Keys are quantity names from the template; values are strings
⚠
All property values must be strings, even numbers: '5000' not 5000. Properties not specified take template defaults.
python — create an unconfined groundwater cell
system.create_block(
    name='cell(1:1)',
    type='Unconfined Groundwater cell',
    properties={
        'area':                  '5000',   # m²
        'base_elevation':        '0',      # m
        'hydraulic_conductivity': '5',      # m/day
        'piezometric_head':      '10.0',   # m (initial)
        'porosity':              '0.3',
        'specific_yield':        '0.2',
        'recharge_per_area':     'Recharge.csv',  # resolved from working_folder
    }
)

# Fixed-head boundary
system.create_block('fixed_head_left', 'fixed_head',
                    {'head': '10', 'Storage': '100000'})

Finalize
Must be called after all blocks and links are created, and before solve().
finalize()
build
Wires up the model graph and prepares it for simulation. Internally calls:
  1. SetAllParents() — links all objects to their parent system
  2. SetQuanPointers() — resolves cross-references between quantities
  3. CalcAllInitialValues() — computes initial state from expressions
  4. InitiatePrecalculatedFunctions() — prepares time-series lookups
⚠
Do not call set_block_property or set_link_property after finalize() — the solver uses internal state that was set during finalization. Property changes after this point are not reflected.
python
system.finalize()
system.solve()
Parameters
Parameters define calibration variables that can be mapped to block/link properties. Used for inverse modeling, sensitivity analysis, and RL action spaces.
append_parameter(name: str, low: float, high: float, prior_distribution: str = "normal") → bool
param
Registers a calibration parameter with a name, feasible range, and prior distribution type.
ParameterTypeDescription
namestrUnique parameter name (referenced in set_as_parameter)
lowfloatLower bound of feasible range
highfloatUpper bound of feasible range
prior_distributionstr"normal" or "lognormal"
python
system.append_parameter('K', low=2.0, high=10.0, prior_distribution='normal')
set_as_parameter(object_name: str, quantity: str, parameter_name: str)
param
Links a block or link quantity to a parameter. The parameter value will override the quantity value when apply_parameters() is called.
python — assign K to hydraulic_conductivity in all cells
system.append_parameter('K', 2.0, 10.0)
for i in range(1, n+1):
    system.set_as_parameter(f'cell({i}:1)', 'hydraulic_conductivity', 'K')
set_parameter_value(name: str, value: float)
param
Sets the current value of a parameter. Combined with apply_parameters(), this is how you update model inputs during calibration or RL episode steps.
apply_parameters()
param
Propagates current parameter values to all linked quantities. Call before solve() during calibration loops.
python — calibration loop pattern
for K_val in candidate_values:
    system.set_parameter_value('K', K_val)
    system.apply_parameters()
    system.solve(apply_parameters=False)  # already applied
    obj = system.get_objective_function_value()
get_objective_function_value() → float
param
Returns the current scalar objective function value (e.g. sum of squared residuals vs observations). Returns 0 if no objective function is defined.
Solve & Output
solve(apply_parameters: bool = True, uniformize_output: bool = True)
solve
Runs the full time integration from simulation_start_time to simulation_end_time using the implicit Newton-Raphson solver.
ParameterTypeDefaultDescription
apply_parametersboolTrueApply parameter values before solving
uniformize_outputboolTrueInterpolate outputs to uniform time intervals
ℹ
Solver progress and timestep information is printed to stdout unless set_silent(True) was called.
write_outputs()
solve
Writes simulation results to the output file (path set by alloutputfile system setting, relative to working_folder). The output file contains time series for all quantities marked "includeinoutput": true in the template.
python
system.finalize()
system.solve()
system.write_outputs()
print('Output written to:', system.working_folder() + 'output.txt')
Save & Load
Models can be saved as .ohq script files (human-readable, reloadable) or as .json files (structured, includes post-solve state).
save_to_script(filename: str, templatefilename: str = "", addedtemplates: list[str] = []) → bool
io
Saves the model as an .ohq script file. The file can be reopened in the OpenHydroQual GUI or loaded via create_from_script. Post-solve state (final piezometric heads, etc.) is included.
ParameterTypeDescription
filenamestrOutput path for the .ohq file
templatefilenamestrMain template filename (empty = use addtemplate only)
addedtemplateslist[str]List of additional template paths to record in the file header
python
RESOURCES = '/path/to/resources/'
system.save_to_script(
    WORKDIR + 'model_saved.ohq',
    '',
    [RESOURCES + 'main_components.json',
     RESOURCES + 'unconfined_groundwater.json']
)
save_to_json(filename: str, addedtemplates: list[str] = [], allvariables: bool = False, calculatevalue: bool = False) → bool
io
Saves the model state as a structured JSON file. Useful for programmatic inspection of model state.
ParameterTypeDefaultDescription
filenamestr—Output path for the JSON file
addedtemplateslist[str][]Template names to record
allvariablesboolFalseInclude all quantities (not just ask_user=true)
calculatevalueboolFalseEvaluate expressions and include computed values
python — save full post-solve state
system.save_to_json(
    WORKDIR + 'model_full.json',
    addedtemplates=['main_components.json', 'unconfined_groundwater.json'],
    allvariables=True,
    calculatevalue=True
)

Full Example: 1D Groundwater
Complete working example — 10-cell 1D unconfined aquifer with fixed-head boundaries.
A
1D Unconfined Groundwater Model

10 cells · fixed-head boundaries · hydraulic conductivity calibration parameter

import openhydroqual_py as ohq

RESOURCES = '/home/user/OpenHydroQual/resources/'
WORKDIR   = '/home/user/projects/groundwater/'

# ── Setup ────────────────────────────────────────────────────────
system = ohq.System()
system.set_default_template_path(RESOURCES)
system.set_working_folder(WORKDIR)
system.set_silent(False)

system.load_template(RESOURCES + 'main_components.json')
system.add_template(RESOURCES + 'unconfined_groundwater.json')

# ── Simulation settings ──────────────────────────────────────────
system.set_system_value('simulation_start_time', '0')
system.set_system_value('simulation_end_time',   '90')
system.set_system_value('initial_time_step',     '0.01')
system.set_system_value('write_interval',        '100')

# ── Aquifer cells ────────────────────────────────────────────────
n = 10
for i in range(1, n + 1):
    head = 10.0 - (i - 1) * (10.0 - 9.0) / n
    system.create_block(
        name=f'cell({i}:1)',
        type='Unconfined Groundwater cell',
        properties={
            'area':                  '5000',
            'base_elevation':        '0',
            'hydraulic_conductivity': '5',
            'piezometric_head':      str(round(head, 5)),
            'porosity':              '0.3',
            'recharge_per_area':     WORKDIR + 'Recharge.csv',
            'specific_yield':        '0.2',
        }
    )

# ── Fixed-head boundaries ────────────────────────────────────────
system.create_block('fixed_head_left',  'fixed_head', {'head': '10', 'Storage': '100000'})
system.create_block('fixed_head_right', 'fixed_head', {'head': '9',  'Storage': '100000'})

# ── Internal links ───────────────────────────────────────────────
for i in range(1, n):
    system.create_link(
        name=f'link({i}:{i+1})',
        type='UC_groundwater_link',
        from_block=f'cell({i}:1)',
        to_block=f'cell({i+1}:1)',
        properties={'length': '50', 'width': '100'}
    )

# ── Boundary links ───────────────────────────────────────────────
system.create_link('link_left',  'unconfined_groundwater_to_fixedhead',
                   'cell(1:1)',    'fixed_head_left',
                   {'length': '50', 'width': '100'})
system.create_link('link_right', 'unconfined_groundwater_to_fixedhead',
                   f'cell({n}:1)', 'fixed_head_right',
                   {'length': '50', 'width': '100'})

# ── Finalize and solve ───────────────────────────────────────────
system.finalize()
system.solve()
system.write_outputs()

# ── Save ─────────────────────────────────────────────────────────
system.save_to_script(
    WORKDIR + 'model_saved.ohq', '',
    [RESOURCES + 'main_components.json',
     RESOURCES + 'unconfined_groundwater.json']
)
system.save_to_json(
    WORKDIR + 'model_saved.json',
    addedtemplates=['main_components.json', 'unconfined_groundwater.json']
)
print('Done')

Save a programmatically-built model as an .ohq file, then reload and re-solve it.

Round-trip: Build → Save → Reload → Solve

Verify that saved scripts are valid and produce identical results

import openhydroqual_py as ohq

RESOURCES = '/home/user/OpenHydroQual/resources/'
WORKDIR   = '/home/user/projects/groundwater/'

# ── Build and save (as above) ────────────────────────────────────
system = ohq.System()
# ... (build model as in Example A) ...
system.finalize()
system.solve()
system.save_to_script(
    WORKDIR + 'model.ohq', '',
    [RESOURCES + 'main_components.json',
     RESOURCES + 'unconfined_groundwater.json']
)

# ── Reload and re-solve ──────────────────────────────────────────
system2 = ohq.System()
system2.set_default_template_path(RESOURCES)
system2.set_working_folder(WORKDIR)
system2.create_from_script(
    WORKDIR + 'model.ohq',
    RESOURCES + 'settings.json'
)
system2.solve()
system2.write_outputs()
print('Reload complete')
